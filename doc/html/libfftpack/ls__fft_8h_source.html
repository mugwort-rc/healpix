41 /*!\defgroup fftgroup FFT interface
42 This package is intended to calculate one-dimensional real or complex FFTs
43 with high accuracy and good efficiency even for lengths containing large
44 prime factors.
45 The code is written in C, but a Fortran wrapper exists as well.
46 
47 Before any FFT is executed, a plan must be generated for it. Plan creation
48 is designed to be fast, so that there is no significant overhead if the
49 plan is only used once or a few times.
50 
51 The main component of the code is based on Paul N. Swarztrauber's FFTPACK in the
52 double precision incarnation by Hugh C. Pumphrey
53 (http://www.netlib.org/fftpack/dp.tgz).
54 
55 I replaced the iterative sine and cosine calculations in radfg() and radbg()
56 by another algorithm, which slightly improves the transform accuracy for
57 real FFTs with lengths containing large prime factors.
58 
59 Since FFTPACK becomes quite slow for FFT lengths with large prime factors
60 (in the worst case of prime lengths it reaches \f$\mathcal{O}(n^2)\f$
61 complexity), I implemented Bluestein's algorithm, which computes a FFT of length
62 \f$n\f$ by several FFTs of length \f$n_2\ge 2n-1\f$ and a convolution. Since
63 \f$n_2\f$ can be chosen to be highly composite, this algorithm is more efficient
64 if \f$n\f$ has large prime factors. The longer FFTs themselves are then computed
65 using the FFTPACK routines.
66 Bluestein's algorithm was implemented according to the description on Wikipedia
67 (<a href="http://en.wikipedia.org/wiki/Bluestein%27s_FFT_algorithm">
68 http://en.wikipedia.org/wiki/Bluestein%27s_FFT_algorithm</a>).
69 
70 \b Thread-safety:
71 All routines can be called concurrently; all information needed by
72 <tt>ls_fft</tt> is stored in the plan variable. However, using the same plan
73 variable on multiple threads simultaneously is not supported and will lead to
74 data corruption.
75 */
