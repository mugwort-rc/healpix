Upgrading from 1.2 to 2.0

The internal structure of release 2.0 is quite different from release 1.2 and to avoid confusion during the compilation we highly recommend to put the new release in a different directory, rather than putting the new package on top of the old one. If you actually change the name of the 'active' HEALPix directory care must be taken that all references to the old directory are removed from your system profile before adding the new ones (see Note on Re-installation).
